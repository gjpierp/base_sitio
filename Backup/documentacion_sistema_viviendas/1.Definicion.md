1. Resumen de la Solución Proyectada
   El sistema "db_viviendas_fiscales" permitirá:
   Gestión de Viviendas y Pabellones: Control individual y compartido (piezas/pabellones).
   Asignación por Puntaje: Cálculo diferenciado para Oficiales y Personal de Cuadro Permanente (PCP).
   Integración Digital: Obtención de UF en tiempo real, facturas de servicios y vinculación con SIAP/SISREM.
   Automatización Documental: Almacenamiento y generación de contratos mediante IA/Plantillas.
   Trazabilidad Total: Historiales de ocupación, pagos y cambios de estado. Los pagos deben considerar porcentaje, fijo, UF

5. API Endpoints (Estructura de Integración)
   Para la integración con SIAP y SISREM, se deben implementar los siguientes endpoints bajo estándar REST:
   integracion con empresas de servicios basicos, se deben implementar los endpoints necesarios para cada caso.
   GET /api/v1/integracion/siap/{rut}: Obtiene datos del personal y cargas familiares.
   POST /api/v1/integracion/sisrem/descuento: Envía la instrucción de descuento mensual de arriendo/seguros.
   GET /api/v1/externo/uf: Conexión con el Banco Central o SBIF para actualización diaria de la UF.
   POST /api/v1/viviendas/asignar: Ejecuta la lógica de negocio de asignación y genera el contrato automático.

6. Consideraciones Adicionales (Lo que faltaba)
   Módulo de Mantención de Usuarios: Se deben crear las tablas vvff_usuarios, vvff_roles y vvff_permisos para controlar quién puede editar los puntajes o validar las facturas de servicios.
   Auditoría (Trazabilidad): Se recomienda una tabla vvff_auditoria que registre usuario_id, accion, tabla_afectada y timestamp para cumplir con normativas de transparencia.
   Alertas: Un sistema de notificaciones para cuando una vivienda desocupada supere un umbral de gasto en servicios básicos.
   Para acceder a plantillas legales y normativas actualizadas, puede consultar el portal de Transparencia FACH.
